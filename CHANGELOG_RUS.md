# Changelog (История изменений)

## v0.13.92
- **ИСПРАВЛЕНО:** Всплывающее уведомление "Refreshing..." (toast через `answerCallbackQuery`) могло молча падать на медленных роутерах. Переменная `CB_ANSWER_TEXT` задавалась до выполнения `clash_request`, но сам ответ отправлялся только после завершения `handle_command`. К этому моменту таймаут Telegram (~5 сек) мог истечь, вызывая ошибку "Bad Request: query is too old" и отсутствие уведомления. **Фикс:** В `proxy_menu` функция `answer_callback` теперь вызывается немедленно (до запроса к clash), после чего ставится флаг `CB_ANSWER_TEXT="__ANSWERED__"`, чтобы главный цикл пропустил повторную отправку ответа. Теперь уведомление появляется мгновенно по клику.

## v0.13.90
- **ИСПРАВЛЕНО:** `get_selector_tag()` использовал слишком жадный fallback, который мог украсть данные из другой секции. Если активная секция была "antiz" в режиме `urltest`, fallback искал ЛЮБОЙ узел типа Selector во всем ответе Clash API (игнорируя привязку к секции) и брал самый большой — им оказывался селектор от "main". Итог: бот показывал прокси и маршрут от main, пока пользователь находился в antiz. **Трехэтапный фикс:** (1) добавлена явная проверка тега `$sec+"-out"` перед фоллбеком; (2) fallback теперь принимает узлы `URLTest` помимо `Selector` (чтобы находить секции в авто-режиме); (3) fallback отфильтрован — ищет только те узлы, чьи ключи начинаются с имени активной секции. Кража данных между секциями невозможна.
- **НОВОЕ:** Защита переключения в режим URL. Переключение в режим URL без настроенной `proxy_string` больше не вызывает перезапуск `podkop` (что привело бы к фатальному крашу "[fatal] Proxy string is not set"). Бот сохраняет режим в UCI, задерживает reload и переходит в статус `wait_url_link`. Пользователь отправляет ссылку в чат, бот сохраняет её и только потом безопасно перезапускает службу. Кнопка Cancel отменяет действие без перезапуска.
- **НОВОЕ:** Предупреждение о режиме URL на экране переключения. Если `proxy_string` пуст, экран подтверждения показывает красное предупреждение о том, что нужно будет ввести ссылку до перезапуска. Если ссылка уже есть, экран показывает текущий URL.
- **НОВОЕ:** Check D (watchdog изменения leaf-узла) теперь отличает ручное переключение от автоматического. Отслеживает поле `.now` в Clash API (последний выбор селектора) вместе с конечным (resolved) сервером. Если изменился `.now` → алерт пишет "Proxy manually switched" (пользователь переключил сам через бота или LuCI). Если `.now` не менялся, но конечный сервер сменился → алерт пишет "Proxy auto-switched" (URLTest сам выбрал сервер с лучшим пингом). Это убирает путаницу, когда бот докладывал о переключении, которое юзер только что сделал сам.
- **ИСПРАВЛЕНО:** `get_active_proxy_name()` не разворачивал группы URLTest — возвращал сырой тег (например, "main-urltest-out") вместо реального активного прокси. Раньше искал только тип Selector; теперь использует `_resolve_leaf()`, который проходит по всей цепочке Selector/URLTest/Fallback/LoadBalance. Исправлен баг "Active Route: main-urltest-out" в главном меню.
- **ИСПРАВЛЕНО:** Отображение версии `podkop` склеивало две версии (например, "v0.7.10-r1 v0.7.14-r1") на кастомных форках OpenWrt вроде RouteRich, где `opkg` отдает несколько строк `Version:`. Добавлен `tail -1` во все 5 мест проверки версии — теперь берется только последняя (самая свежая) запись.
- **ИСПРАВЛЕНО:** Счетчик `proxy_link` (используется для кнопки Clone) показывал неверные данные на некоторых версиях OpenWrt, где `uci show` выводит все элементы списка в одну строку, а `grep -c` считал строки вместо элементов. Заменено на `eval "set -- $raw"; count=$#` — работает независимо от формата вывода.
- **ИНТЕРФЕЙС:** Заголовок карточки `proxy_menu` теперь отражает реальный режим: "Outbound Selector" для ручного режима, "URLTest Outbounds" для авто-режима.
- **ИНТЕРФЕЙС:** Режим URL переименован повсеместно для большей ясности: в главном меню "URL Links" → "Single URL"; заголовок "URL Links" → "Single URL Proxy"; кнопка "Add Link" → "Set URL".
- **ИНТЕРФЕЙС:** Добавлена опция №4 (Uninstall) в скрипт установки — полное удаление с двойным подтверждением (ввести "YES", затем "REMOVE"). Удаляет бинарник бота, init.d скрипт, `/etc/config/podkop_bot` и все `/tmp` файлы рантайма. Сам `podkop` и `sing-box` остаются нетронутыми.
- **ИСПРАВЛЕНО:** КОРЕНЬ ПРОБЛЕМЫ с циклом ложных срабатываний watchdog (баг "tir1"). Логи показали: `route_key_raw='tier1' hex= clean=tir1`. Причина: утилита `tr` в BusyBox на OpenWrt 24.10.x ошибочно считает букву 'e' (0x65) пробельным символом (`[:space:]`). Подтверждено на Redmi AX6000 (stock OpenWrt 24.10.5). В итоге `tr -d '[:space:]'` превращал "tier1" в "tir1", что не попадало под маску валидных маршрутов `tier1|tier2_*` → watchdog каждые 120 сек слал IPC сигнал на рестарт, спамя "recover old=unknown" в лог. **Фикс:** все `tr -d '[:space:]'` заменены на `tr -d '\n\r\t '` (только явные пробелы и переносы). 
- **УДАЛЕНО:** Отладочный логгер `[RouteWrite]` из `_write_main_route` (добавлен в v0.13.89, спамил на каждый вызов API).
- **УДАЛЕНО:** Отладочный логгер `[Watchdog] route_key_raw/hex`.
- **РЕФАКТОРИНГ:** Все системные логи и алерты в Telegram переписаны для удобного чтения. Syslog: дампы внутренних переменных (RECOVERY_MODE, FAST/POLL) заменены на читаемые предложения. Строки кэша переименованы в `[Core]`. Сводка здоровья сжата в одну строку `[Health]`. Watchdog IPC логи описывают действия, а не механику. Алерты в Telegram теперь сфокусированы на том, как проблема влияет на пользователя, убраны технические термины (tier1, control plane, Scope:) в пользу простого английского.

## v0.13.89
- **ИСПРАВЛЕНО:** Удаление прокси (`ask_del_px_*`) — найдена и устранена первопричина. Функция `get_selector_link_by_index` предполагала, что тег "main-N-out" соответствует позиции N-1 в списке UCI. Это не так: после любого удаления нумерация тегов в sing-box расходится со списком UCI. Неверный индекс → неверная ссылка → `uci del_list` молча удалял ДРУГОЙ прокси или вообще ничего. Привязка к индексу полностью убрана. **Новый подход:** всегда ищет совпадение по `server:port` из `TAG_URI_CACHE` (не зависит от порядка, уникален). Два уровня фоллбека: поиск в `UCI_LINKS_CACHE` и сканирование "живого" `uci show`. Покрывает все типы протоколов (vless, hy2, trojan, tuic, ss+auth, ss-no-auth). Ошибка теперь возвращает на экран `px_view_N` вместо корневого меню.
- **ИСПРАВЛЕНО:** `extract_server_port_from_uri()` падал на ссылках shadowsocks без пароля (формат `ss://host:port` без `@`). Новая регулярка корректно парсит оба варианта: с авторизацией и без.
- **ИСПРАВЛЕНО:** `test_px_` (кнопка Test в карточке прокси) заменяла саму карточку сообщением о загрузке, вызывая визуальный прыжок интерфейса. Теперь статус проверки отправляется отдельным сообщением ниже карточки и удаляется после завершения.
- **ИСПРАВЛЕНО:** Аналогичный "прыжок" при `cmd_all_delay_test`. Также теперь сохраняется номер текущей страницы при обновлении меню.
- **ИСПРАВЛЕНО:** Кнопка Refresh (`proxy_menu_p_N`) выглядела зависшей на ~10 сек без обратной связи. Теперь сразу показывает индикатор "Refreshing..." редактируя саму карточку до вызова API. Добавлена логика повторных попыток при пустом ответе + улучшенное сообщение об ошибке с кнопкой Retry.

## v0.13.88
- **ИСПРАВЛЕНО:** В `clash_request()` отсутствовал флаг `--max-time` для `curl`. Флаг `--connect-timeout 3` покрывает только TCP-хендшейк; если API принимало соединение, но висело (например, из-за нехватки памяти), watchdog наглухо блокировался. Добавлено `--max-time 10` во все вызовы curl.
- **ИСПРАВЛЕНО:** Накопление зомби-процессов от Watchdog при аптайме (30+ дней). Сабшеллы `probe_all_socks_write` оставались зомби, пока родитель не вызовет `wait()`. **Фикс:** теперь сохраняется `_last_probe_pid` и вызывается `wait` перед запуском следующей проверки.
- **ИСПРАВЛЕНО:** Утечки временных файлов в `refresh_public_ip_cache()` при нехватке RAM (tmpfs full). Все вызовы `mktemp` теперь имеют проверку на ошибку и явный откат (удаление директории блокировки). Команда `mv` заменена на `mv || rm -f "$tmp"`.
- **ИСПРАВЛЕНО:** В `probe_all_socks_write()` добавлена защита от сбоя `mv`, чтобы временный файл не оставался висеть на диске.
- **ИСПРАВЛЕНО:** Баг составного условия в механизме reprobe для tier4/tier5. Конструкция `A || { B && C }` в некоторых версиях ash работает неоднозначно. Заменено на явные `if/elif/else` блоки.
- **ИСПРАВЛЕНО:** Очистка `trap` не захватывала все временные файлы. Добавлены шаблоны: `/tmp/podkop_updates.*`, `/tmp/podkop_req.*`, `/tmp/podkop_clash.*` и др.
- **НОВОЕ:** Жесткая очистка `/tmp` при старте. Бот сканирует `/tmp` на наличие старых файлов `podkop_*` (старше 60 сек) и удаляет артефакты от предыдущих сессий, убитых по `kill -9`.
